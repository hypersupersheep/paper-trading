# 协作交接

各 agent 之间**交接任务、汇报进度、共享记忆**的唯一公共文件。放在 app 仓库根,所有挂了本仓库的 agent 都能直接读写。

---

## 署名制度

- 每条记录**必须署名**:谁做的就署谁的代号,例:`— QR-A`。
- 格式:`### [日期] 标题 — 代号`,正文写「做了什么 / 给对方的需求 / 待确认」。
- 给别人派活/提需求,在条目里写清「**@对方代号:需要你做…**」;对方完成后**在同一文件追加回执并署名**。
- 不删别人的记录(append-only);要更正就追加一条说明。

### [2026-06-28] 拔除 sleeve 资金单元,改单一 account.cash 模型(API v1→v2,破坏性)— QR-A

**改动**:同事反馈 sleeve(资金单元)没用且增加难度。**连根拔起**:每个账户回到**单一现金池(account.cash)+ 一组持仓**,不再有 per-strategy 资金桶。

**影响面(均已改完、188 测试全绿、E2E 冒烟通过)**:
- 后端:`trading_store` 现金/持仓/下单/补录/逆回购全走 `account.cash`;`audit_store` 去 `sleeve_id`(历史列保留,新行不写);`risk_store` 改账户级(`max_sleeve_exposure`→`max_exposure`,删 sleeve 级配置);strategy/timing/scheduler stores + workers 去 sleeve。
- **API(破坏性,故 `API_VERSION` 1→2)**:删 `POST /api/accounts/{id}/sleeves`、`/api/sleeves/{id}/active`、`/api/sleeves/{id}/allocation`;`place_order`/`backfill`/`run`/`bind`/`risk` 不再收 `sleeve_id`;`meta.capabilities` 去掉 `sleeves`。
- **组合概览 JSON(Admin 关注)**:account 不再有 `sleeves`/`unallocated_cash`/`allocated_cash`/`sleeve_cash`;改为 `cash`(并保留 `total_cash` = cash 作兼容)。position 去掉 `sleeve_id`/`sleeve_name`/`strategy_id`。
- **老库自动迁移**:`account.cash = unallocated_cash + Σ sleeve.available_cash`;持仓按 `(account,symbol)` 合并;`paper_orders`/`strategy_runs`/`scheduler_tasks` 去 `sleeve_id`;丢 `sleeves` 表。迁移在 `foreign_keys=OFF` 独立事务里跑,可重入。本机真实库已验证迁移干净。
- 前端、agent SDK/CLI、SKILL.md 同步去 sleeve。版本 **v1.15.0**。
